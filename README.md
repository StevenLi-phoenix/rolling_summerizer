# Novel Summarizer with Rolling Window (Local LLM)

A Python-based novel summarization tool that uses local Large Language Models (LLMs) via LMStudio to automatically generate chapter summaries from Chinese novels. The tool employs a rolling window approach to handle long texts that exceed LLM context limits.

## Features

- 🔍 **Automatic Chapter Detection**: Supports multiple chapter title patterns (第N章, Chapter N, etc.)
- 🪟 **Rolling Window Processing**: Handles long chapters by splitting them into overlapping chunks
- 🤖 **Local LLM Integration**: Uses LMStudio with qwen3-8b or other local models
- 📝 **Multiple Encodings**: Supports UTF-8, GBK, GB2312, and Big5 text encodings
- 📊 **Token Management**: Accurate token counting using tiktoken
- 📄 **Markdown Output**: Generates well-formatted summary files
- 🏠 **Privacy-First**: All processing happens locally, no data sent to external APIs

## Setup

### 1. LMStudio Setup

1. **Download and Install LMStudio** from [https://lmstudio.ai/](https://lmstudio.ai/)

2. **Download a Model** (recommended: Qwen models):
   - Open LMStudio
   - Go to the "Discover" tab
   - Search for "Qwen" and download Qwen3-8B or similar
   - Wait for the download to complete

3. **Start the Local Server**:
   - Go to the "Local Server" tab in LMStudio
   - Select your downloaded model (e.g., Qwen3-8B)
   - Click "Start Server"
   - Server will typically start on `http://localhost:1234`

### 2. Python Environment Setup

Using conda (as you specified):

```bash
# Activate conda base environment
conda activate base

# Install dependencies
pip install -r requirements.txt
```

### 3. Configure Settings (Optional)

Create a `.env` file if you want to customize settings:
```
# Optional: Override default server URL
LMSTUDIO_URL=http://localhost:1234

# Optional: Override default model name
MODEL_NAME=qwen3-8b
```

## How It Works

### 1. Chapter Detection

The tool uses regex patterns to identify chapter boundaries:
- `第\d+章[^\n]*` - Matches "第1章 标题"
- `Chapter\s+\d+[:\s][^\n]*` - Matches "Chapter 1: Title"
- `第[一二三四五六七八九十百千万]+章[^\n]*` - Matches "第一章 标题"

### 2. Rolling Window Processing

For chapters exceeding the model's context limit:

1. **Calculate chunk size**: `chunk_size = max_tokens - safety_buffer`
2. **Set stride**: `stride = chunk_size // 2` (50% overlap)
3. **Split into chunks**: Create overlapping text segments
4. **Generate summaries**: Summarize each chunk individually via local LLM
5. **Aggregate**: Combine chunk summaries into a coherent chapter summary

### 3. Local LLM Communication

The tool uses OpenAI-compatible API that LMStudio provides:
```python
# Example configuration for Qwen3-8B:
max_tokens = 8000
safety_buffer = 500
chunk_size = 7500
stride = 3750
```

## Configuration

### Chunk Size Optimization

You can adjust chunking parameters in `main.py`:

```python
class NovelSummarizer:
    def __init__(self, model='qwen3-8b', max_tokens=8000):
        self.buffer = 500  # Safety buffer
        self.chunk_size = max_tokens - self.buffer
        self.stride = self.chunk_size // 2  # 50% overlap
```

## Performance Tips

1. **Use models with larger context** (e.g., Qwen3-14B) to reduce chunking overhead
2. **Adjust chunk overlap** - more overlap for better coherence, less for faster processing
3. **GPU acceleration** - Enable GPU in LMStudio for faster processing
4. **Model selection** - Balance between model size and processing speed
5. **Batch processing** - Process multiple novels sequentially

## Troubleshooting

### Common Issues

1. **"Failed to connect to LMStudio server"**
   - Make sure LMStudio is running and server is started
   - Check if the server URL is correct (default: http://localhost:1234)
   - Verify that a model is loaded in LMStudio

2. **"Model 'qwen3-8b' not found"**
   - The tool will automatically use the first available model
   - Make sure you have downloaded and loaded a model in LMStudio

3. **Slow processing**
   - Enable GPU acceleration in LMStudio if you have a compatible GPU
   - Use smaller models for faster processing
   - Reduce chunk overlap for speed

4. **Out of memory errors**
   - Use smaller models (e.g., 7B instead of 14B)
   - Reduce max_tokens parameter
   - Close other applications to free up RAM

5. **Encoding errors**
   - The tool tries multiple encodings automatically
   - For special encodings, modify the `load_novel()` function
